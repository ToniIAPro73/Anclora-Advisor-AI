# QA Gate Baseline

## Criterios NO-GO

- Errores en `lint`, `type-check` o `build`.
- Regresion funcional en `/api/chat`.
- Errores P0 en chat UI.
- Fugas de datos sensibles en logs o respuestas.

## Cierre GO

- Checks en verde.
- Contrato API verificado.
- Evidencia documentada en `sdd/features/<feature>/`.
